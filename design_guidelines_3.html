AR Assembly Design Guidelines


Version 3

  •    Shorten verbose audio
  •    Supplement audio with visuals
  •    Present relevant virtual content
  •    Superimpose on the real world
  •    Depict action with 3D elements
  •    Clearly depict hand maneuvers
  •    Use salient cues
  •    View-stabilize critical info
  •    Keep text minimal
  •    Choose FOV based on workspace
  •    Support hands-free interactions

Shorten or replace verbose audio instructions with visual references when explaining specific object configurations.

 

Example

Supplement audio instruction with visual aids when audio-only communication is not effective (e.g. for tasks that take place in a noisy environment)

 

Example  

Only present virtual object parts of the assembly object (rather than a complete virtual replica) that are needed for the current step so that relevant information is immediately apparent.

 

Example

For intuitive instructions, superimpose virtual content over real-world objects, rather than off-to-the-side, but be wary of occlusion.

 

Example

When depicting action sequences, use animated 3D objects rather than static 2D diagrams.

 

Example

Rather than using abstract entities (e.g. arrows) to depict specific hand maneuvers, use virtual hands or embed a demo video to clearly demonstrate the procedure.

 

Example

Use salient attentional visual indicators (e.g. high contrast color highlights) to help users quickly detect and identify targets in the real environment.

 

Example

View-stabilize critical information (e.g. a safety warning) so that it is visible to the user regardless of where the device is pointed.

 

Example

Only use text when it is strictly required (e.g. when the user must learn the names of specific assembly components). Keep the text short and simple so that it does not clutter the screen or overwhelm the user.

 

Example

Choose display hardware that can fit the assembly workspace and virtual content within its FOV. This will reduce excessive head movements for HMD and panning operations for HHD .

Example

Support hands-free interaction techniques (e.g. voice commands) so that the user can use both hands during the assembly task.
